# Program audit — benchmark import P0–P6 (2026-07-10)

Six audit lenses (published-artifact integrity ×2, licenses/SA-firewall, union-holdout,
registry+adversarial-overlap, builders, docs/claims) ran as independent agents against primary
files and the published HF artifacts; the holdout lens and the per-finding skeptic passes were
re-executed inline after a spend-limit interruption. Every major finding below was verified by
execution before being fixed. Coverage inventories live in the audit transcript
(session scratchpad `audit/`).

## Verified clean (the positives)

- **All 7 frozen catalogs**: published bytes == local; counts exact; gt_kind semantics hold for
  all 169 problems; expression records literal + aligned + engine-reproducing y_reference;
  resolver sha256 verified on fresh downloads. The only exceptions are float32-quantization
  artifacts on the cp3 GW toys (justified, ≤1 ulp for exact entries; documented below).
- **All declarative catalogs**: counts exact, schema conformant, realize cleanly (full for
  ≤200-entry catalogs; seeded 300-samples for the two large ones) — with the single fastsrb@1
  exception fixed below.
- **Union holdout**: registration of all GT-bearing catalogs yields 6,150 structures + 5,828
  images in ~20 s; membership verified for every catalog (zero-delta registrations are ordering
  artifacts, not gaps); black-box catalogs contribute nothing, silently, as designed.
- **Builders**: all 11 byte-deterministic in a clean worktree (the dso/srsd-dummy pair
  reproduces committed artifacts only together with the audit-tool pass — now documented in
  both builders).
- **Registry**: rebuild byte-identical to committed; explicit links recounted correct;
  no SA expression text in the registry (names only).
- **SA firewall**: no OEIS-derived content outside assets_sa/; SA HF repo card + LICENSE
  correct; main manifest does not list SA catalogs; per-entry attribution present.

## Accepted as-is (justified, documented)

- cp3-cosmo C6a/b/c float64-render deviations vs stored y_reference (≤5.8e-3 rel on
  zero-crossing rows): float32-x quantization of byte-faithful upstream data; GT self-render
  FVU ~1e-13, six orders below the strict recovery gate.
- cp3-cosmo exact entries with y vs y_ref differing by exactly 1 float32 ulp on some rows:
  consistent with machine-precision freezing.
- v23-val realizes config/seed-dependently (~743/1000 at one probe config) via the honest
  placeholder protocol — the 1000 count is the skeleton-pool size (standing placeholder-filter
  policy applies).
- Registry clusters conflate exact duplicates with same-skeleton/different-literal families —
  correct under the declared masked-literal identity; consumers must not "keep one per
  cluster" for eval purposes.
